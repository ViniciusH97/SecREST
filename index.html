Relatório Técnico de Auditoria de Segurança em API RESTful

Cliente: Exemplo S/A
Data da Auditoria: 31/07/2025
Analista Responsável: ViniciusH97
Metodologia: OWASP API Security Top 10

1. Sumário Executivo

Durante a auditoria da API https://api.exemplo.com, foram identificadas 12 vulnerabilidades, sendo 3 críticas, 4 altas, 5 médias e 0 baixas.

Ação Imediata Requerida: A SecREST recomenda mitigação imediata das falhas classificadas como críticas e altas.

2. Resumo de Vulnerabilidades

Severidade Quantidade Exemplos
CRÍTICA 3 Broken Authentication, JWT Insecure
ALTA 4 SQL Injection, Rate Limit Bypass
MÉDIA 5 CORS, Info Leakage, Verbose Errors
BAIXA 0 —

3. Principais Falhas Encontradas

3.1 [CRÍTICA] Broken Authentication – Login Endpoint
Descrição: Falta de verificação adequada em tokens e sessões.
Endpoint: POST /api/v1/login
Impacto: Permite que um atacante force logins com brute-force.
Recomendações:
  • Implementar rate limiting e lockout por IP
  • Adotar autenticação multifator (MFA)
  • Validar tokens com expiração e assinatura
3.2 [ALTA] SQL Injection – Parâmetro Não Saneado
Endpoint: /api/v1/users?id=123 OR 1=1
Exploração: Manipulação do parâmetro id permite bypass em queries SQL.
Recomendações:
  • Utilizar ORMs seguros ou prepared statements
  • Sanitização rigorosa dos inputs de usuário
3.3 [MÉDIA] CORS Misconfiguration
Descrição: Permissão de origens externas não confiáveis.
Header: Access-Control-Allow-Origin: *
Risco: Pode permitir exploração de API por scripts maliciosos.

4. Boas Práticas Encontradas

  • ✓ Uso de HTTPS em todos os endpoints
  • ✓ Tokens JWT com algoritmos robustos (HS256)
  • ✓ Segregação de ambientes (produção, staging)

5. Conclusão

A auditoria evidenciou que, embora a API utilize alguns controles de segurança corretos, há falhas críticas que permitem ataques reais, como brute-force, SQL Injection e exposições CORS.

A aplicação destas correções é prioritária para evitar riscos operacionais, legais e financeiros.
